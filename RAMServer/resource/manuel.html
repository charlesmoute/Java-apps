RAMServer 1.0


L'interface de RAMServer est constitué de quatres grandes parties :

  • Le Menu.
  • Une boîte à outils.
  • Une sortie.
  • Une barre de tâche.
et de trois grand modes :
  • Le mode stop, où le serveur est dans l'état arrêt.
  • Le mode start en multithread; où le serveur est lancé et gére ses connexions clientes en multithreading
  • Le mode start en canaux; où le serveur est lancé et gére ses connexions clientes avec des canaux.

Le menu

Trois menu sont a distinguer :

  • File : regroupant les actions standards de l'application.
  • Edition : regroupant les fonctionnalités supplémentaires de l'application.
  • Help : permettant d'avoir accés a un à propos et à cette aide.


File


Home

Ce menu est constitué des sous-menus suivant :
  • Start : démarre le serveur.
  • Stop : Arrête le serveur
  • Print : imprime la sortie du serveur
  • Exit : Quitte l'application en arrêtant le serveur.


Edition


Home

Ce menu est constitué des sous-menus suivant :
  • Clean Log Folder : Nettoie le dossier des fichiers log.
  • Clean Cache Folder : Nettoie le cache du serveur.
  • Log Files Listing : Donne, au travers d'une fenêtre indépendante,le contenu du dossier des fichiers log.
  • Preference : Donne une fenêtre permettant de paramêtrer l'application.
NB
  • Un nom de fichiers en cache est de la forme name 22012010100215 ce qui correspond à la periode que le fichier de nom name a été mis en cache. Dans notre exemple le fichier name a été mis en cahe le 22/01/2010 à 10:02:15
  • Parmi les préférences on a le choix du mode de gestion des connexions, le numéro de port du serveur, le nombre de connexions simultanénées, zéro pour une infinité, les chemins des dossiers racine du serveur et des fichiers Log. Il faut arrêter le serveur et le relancer pour que les préférences soient prises en compte.


Help


Home

Ce menu est constitué des sous-menus suivant :
  • Manuel : Donne accés à cette aide.
  • About: Donne une fenêtre d'à propos.

La Toolbar

Home

La toolbar est constituée de six icônes, chacune d'elle étant des raccourcies de sous-menus. Les significations de ces icônes sont les suivantes :

  • (Démarre le Serveur; n'est disponible que lorsque le serveur est arrêté)
  • (Arrête le Serveur; n'est disponible que quand le serveur est en cours d'exécution)
  •  (Imprime le contenu de la sortie, et ce qu'importe le mode du serveur)
  • (Donne accés à cette page d'aide)
  • (Affiche une fenêtre permettantde visualiser les fichiers de log)
  • (Ferme l'application)

La Sortie

Home

La sortie est l'écran noire, du milieu. Elle affiche les requêtes que le serveur reçoit ainsi que les réponses de ce dernier à ses clients. Pour Voire les autres fichiers de log, il existe un bouton view, représenté par l'image , qui permet au travers d'une fenêtre de consulter les autres log. Sur la partie edition de cette fenêtre a l'aide du click droit on peut rafraîchir l'écran d'édition.

La barre des tâches.

Home

Elle sert principalement, à afficher trois messages.

  • Un message signalant que le serveur est arrêté.
  • Un message signalant que le serveur gére ses connexions clientes en multithreading.
  • Un message signalant que le serveur gére ses connexions clientes au travers des canaux.
Les deux derniers messages, contrairement au premier, sont affichés quand le serveur est lancé et ceci pour signifier, en permanence, à l'utilisateur le mode de gestion des connexions.
